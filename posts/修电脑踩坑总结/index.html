修电脑踩坑总结

引子

某天夜黑风高，我来到实验室，发现移动鼠标后主机屏幕毫无反应。

换了一个无线鼠标，无效。

赶紧想到了重启大法，开心地按下了重启键，万万没想到，屏幕一片黑了，主机蓝色电源键疯狂闪烁，除此之外，再无反应。

那时单纯的我一定不会想到，未来四天，我会一直栽在这台电脑上，开启我漫漫修电脑的征程……

硬件部分

在询问了老师之后，老师建议我重插内存条。

我哆哆嗦嗦地拆了机箱，在小伙伴们的帮助下，找到并重插了内存条，然而，电源键还是闪烁个不停，屏幕一片黑。

导师驾到后，好心地帮我检查了HDMI接口(CPU主板和1080Ti显卡上两个接口)，两个显示屏，按紧了内存条，检查了风扇，还抠了一个大电容使劲在钥匙在摩擦放电，满怀期待地再次开机，仍然两个显示器一片黑。。不过，唯一的好处是，电源键不再闪烁，而是稳定了。

接下来，导师判断是CPU主板出问题了，叫了个同城达达快递，给送到维修中心去了（疫情封校，可怜的我和好心的小伙伴还把主机抱到了学校门口），不知道为啥，导师让我机械硬盘、移动硬盘、显卡全拆了。维修中心给的反馈是，主机一切正常没有问题，视频输出也很正常，清了个灰又给送回来了，我……

我又满怀希望地把主机抱了回来，结果一插视频线，又没有输出，水土不服，我卒了，就把主机丢在实验室回去睡觉了。第二天一早，导师来巡视后给我打电话，你电脑好了，换了根HDMI的线，一切正常了…我震惊，之前也明明检查过HDMI接口。直至今日，我仍然不知道我的电脑到底出了什么问题，反正经历了重插内存条、放电、清灰等等操作了，就神奇的好了。总之，好了是好事，以结果为导向。

艰难地在实验室大佬的帮助下，又重新插好了机械硬盘和显卡，一开机，两眼一黑，两个问题：1.报错，进不去系统。 2. 电脑发出了神奇的咔哒咔哒响的声音。

大佬拆了好多主机部件后，定位到是风扇的问题，重新组织了布线后，咔哒声消失了。

但是系统报错依然存在:

ACPI Error:[*SB.PCI0.RP*05.PXSX] Namespace lookup failure

软件部分

接上，束手无策后，我遂决定重装系统为Win10（Ubuntu生态感觉不完善，不好用），由于之前电脑的系统为Ubuntu，遇坑不少，记录如下~

1.制作U盘启动盘

（参考https://zhuanlan.zhihu.com/p/36495771）

找了一个文件大小8G（电脑读取实际大小为7.5G）的U盘。 进入微软官方网站下载Win 10的界面，会看到提供的安装工具下载链接。

链接直达：https://www.microsoft.com/zh-cn/software-download/windows10

下载完成后，直接运行Win 10安装工具，可以通过该工具直接升级本电脑至最新版Win 10系统，这里，我们当然选择后面一项：为另外一台电脑安装介质。

然后一直跟着提示走即可，由此制作好U盘启动盘。

2.装系统

开机之后进入BIOS（开机的第一个界面有提示如何进行，我的电脑是华硕主板，所以是按DEL键），不同电脑不一样。

进入BIOS后，找到启动方式，选择刚刚插入的U盘为启动的第一选项，然后重启电脑即可。

接下来就是按照微软的提示，一路往下，值得注意的是，中间遇到一个问题。

提示：windows必须安装在格式化为ntfs的分区。这是由于之前的固态硬盘格式为EXT4（Linux的文件系统），因此我们格式化为NTFS格式即可，步骤如下。（参考https://www.jianshu.com/p/ed7c79865787）

（1）按Shift+F10出现命令提示符输入：

（2）diskpart              @这时要使用管理员权限进入diskpart程序；

（3）list  disk            @查看当前磁盘情况

（4）select  disk    0     @选择需要格式化的磁盘

（5）clean                 @先清除所有磁盘分区

（6）convert    ntfs       @转换成ntfs格式
3.硬盘文件备份

​ 装完系统后，本以为大功告成了……

然后突然想到自己之前为了防止误操作，将机械硬盘给拆了下来，现在重新安装硬盘后，Win10系统无法识别…..在计算机管理 -> 硬盘管理中可以看到该硬盘，格式为EXT4，读取不了，自己一堆文件都在里面。。。

​ Windows和Linux的文件系统不同，linux使用的是ext(Ext2、Ext3、Ext4)，因为Linux支持很多不同的文件系统也包括ntfs，所以Linux系统下一般能直接访问ntfs格式的移动硬盘或者U盘；但是，在Windows下要访问Linux的文件系统比较困难，需要借助第三方软件，Windows目前使用的文件系统主要是ntfs，也支持(FAT、FAT32)。

​ 接连尝试了Ext2 Volume Manager和Ext2Fsd两个软件，由于缺乏教程/有教程仍然无法识别，均宣告失败。

​ 万幸，找到了这篇博客并且在https://blog.csdn.net/qq_32767041/article/details/84069088的指导下，成功对硬盘中的文件进行了备份。

​ Paragon ExtFS for Windows 是一款适用于Windows操作系统的软件，也有 for Mac的，支持读写ext2/ext3/ext4文件系统，操作简单方便，界面简洁美观。更多介绍详见官网：https://www.paragon-software.com/business/extfs-for-windows/。这是一款收费的软件，要120RMB，可以试用10天，网上有破解版的，这里提供一下链接：https://pan.baidu.com/s/1r6Tzga6gOoqLKDaaQW2cRQ。该文件是一个压缩包，里面包含了Paragon ExtFS for Windows 4.2.651和用于破解的注册机v1.2

​ 安装：

  • 将下载的压缩包解压
  • 运行Setup.msi安装Paragon ExtFS，进入软件安装向导，点击next
  • 勾选接受许可协议条款，点击next
  • 根据具体情况选择软件安装路径，点击next
  • 点击install安装
  • 安装完成，勾选试用10天，点击next，点击Finish退出向导

​ 激活：

  • 安装完成后先不要运行软件，解压注册机压缩包，将keygen和另外两个文件放到ExtFS的安装目录下
  • 运行keygen，点击Patch RSA 2048，分别把Paragon Extfs for Windows.exe和extservice.exe这两个文件patch，patch成功后在安装录下会生成两个同名的新exe文件，原文件被自动备份为后缀名为bak的文件
  • 点击Generate生成License，出现提示后点击ok
  • 激活完成，运行软件

安装完Paragon ExtFS for Windows软件后，一打开该软件，在此电脑中就可以看到正常读取的新D盘（EXT4格式的机械硬盘），再拷贝即可顺利备份文件。

4.硬盘格式化

本想直接在此电脑中，右键硬盘进行格式化，不想右键点击格式化完毫无反应，想来应该又是文件系统不同的锅。

于是想到了专业的硬盘管理软件diskgenius，下载diskgenius进行格式化后，却遇到了问题。。。

写扇区错误…不知道这是什么问题。

想到了在硬盘中怎么也删不掉的几个文件后，隐隐约约觉得还是需要专业的EXT4管理软件来做这个格式化的事情。

解决方案：先用Paragon ExtFS for Windows将硬盘格式化为EXT3，清空所有软件，（该软件无法格式化为NTFS格式）。然后再用diskgenius对格式化为EXT3的硬盘再次格式化为NTFS。

至此，顺利解决!
