Hugo+github搭建个人博客

2020年12月18日
| 前端
| 阅读

这是一篇关于使用hugo + github搭建博客的踩坑记录~ 终于赶在2020年的小尾巴上，圆满的完成了个人博客小心愿。

hugo + github搭建博客

Hugo

为什么需要Hugo?

  • 将Markdown文件转换为html的静态网页
# 查看 Hugo 是否安装成功，通过检查版本号
hugo version

## 创建一个新的网站
# 创建工作目录
hugo new site myBlog

#添加一个主题
cd myBlog
git init # 把这个目录变成Git可以管理的仓库：
# 添加子模块
# git submodule add <repository> <path> 
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke


# 在根目录下的 config.toml 文件中添加一行
echo theme = "ananke" >> config.toml

# 添加一个文章
hugo new posts/my-first-post.md

#编译生成静态文件并启动web服务
hugo server -D
#编译生成静态文件
hugo

注意：新Markdown要打开，然后将draft: true改成false，否则默认是草稿形式。 title: “My First Post” date: 2020-12-15T21:32:00+08:00 draft: false

— 执行hugo命令，站点目录下会新建文件夹public/，生成的所有静态网站页面都会存储到这个目录，

— 如果使用Github pages来作为博客的Host，只需要将public/里的文件上传就可以。

Hugo生成的工作目录：

  1. ▸ archetypes/ #包括内容类型，在创建新内容时自动生成内容的配置
  2. ▸ content/ # 网站内容，全部使用markdown格式
  3. ▸ layouts/ # 网站模板文件，决定内容如何呈现
  4. ▸ static/ # 图片、css、js 等静态资源
  5. ▸ themes/ # 存放主题
  6. ▸ config.toml # 是网站的主配置文件

Github Pages

第一次git push

为什么需要Github Pages?

  • Github Pages 本质上是一个静态网站托管系统，你可以使用它为你的每一个仓库制作一个静态网页入口。

它有两种存在方式：

  1. 识别 master branch 根目录下的：README.md 或者 index.html
  2. 识别 master branch 的 /docs 目录下的：README.md 或者 index.html

也就是说：

我们可以把我们的静态网页直接存在 master branch，并在 Github Repository 的 Setting 页中打开 Github Pages 选项，就可以通过一个域名访问到我们的想要的网站了。

很多博客说的都是在根目录下git init…什么的，但是没有说清楚目录，其实结合原理很容易清楚，是在public目录下，生成的所有静态网站页面都会存储到这个目录。

# 进入public目录
cd public 
# 通过git init命令把这个目录变成Git可以管理的仓库：
git init 
# 关联远程库
git remote add origin https://github.com/lvyang0/lvyang0.github.io.git
# 添加文件到本地库
git add .
# 提交文件到本地库
git commit -m "github"
# push到远程库
git push origin master

上传之后我遇到了，个人网站404的情况，这时只需要在public文件夹中新建docs文件夹，再新建txt文档，重命名为CNAME（内容为空即可）。

第二次git push update

在搭建好自己blog网站后，你要在网站上跟新Markdown日志，所以就会第二次git push。 在你的本地静态web上，撰写好md，然后预览，如果没有问题。那么继续push。

hugo #假设你已经完成.md更新
# 第二次生成public
cd public
git init
#填写自己的域名
git remote add origin https://github.com/lvyang0/lvyang0.github.io.git
git add -A
git commit -m "update-hugo"
git push -f origin master

注意，public文件夹，要hugo命令生成，之前第一次的public文件可以rm。 最后浏览器里访问：http://xxx.github.io/，就可以看到更新的日志博客了。

更新博客

主题

完美主义的我渐渐不满足于ananke主题的缺乏归档和分类功能，开始致力于搜寻其他好看的主题。

于是在一通搜寻后看上了，https://github.com/flysnow-org/maupassant-hugo主题。

安装过程在上述链接中都有，均为中文，很是方便。唯一需要特别注意的是，md文件需要放在post而不是posts文件夹中。

代码高亮

安装完主题后，发现一个大问题…

代码没有高亮的功能，不喜欢原生的淡黄色代码底色，喜欢黑底白code+彩色的代码。

保证 Hugo 的版本是高于 v0.65.0 的（很奇怪，我的版本的63也能正常高亮）。

默认的代码高亮配置文件如下，你可以复制到你的配置文件config .toml内对应部分进行修改：

[markup]
  [markup.highlight]
    codeFences = true
    guessSyntax = false
    hl_Lines = ""
    lineNoStart = 1
    lineNos = false
    lineNumbersInTable = true
    noClasses = true
    style = "monokai"
    tabWidth = 4

参考https://dp2px.com/2020/05/29/hugo-highlighting/

目录

该主题支持文章目录（大纲）,大纲是通过h1~h7标题自动生成，在MD文件里就是##这类标题的标记，这里建议最多使用两层文章目录，而且最好是从h2开始，便于SEO优化。 如果要开启一篇文章的文章目录，只需要在config.toml添加toc=true全局开关即可，默认是不开启文章目录的。

亲测这个主题只支持两级标题…即”#“和”##"，不过两级标题大抵也够了。以后文章的唯一大标题就不要了，剩下了大标题和小标题。突然想起来导师给我改开题报告的时候，千叮铃万嘱咐不要有三级标题haha~

文章分类&标签

找了半天资料才发现文章分类很简单，在exampleSite文件夹里面的post中看到的，只需要在每篇文章的前面加上对应的category和tags即可。

例如：

---
title: "Golang Gin 实战（三）| 路由参数"
tags: ["golang"]
categories: ["Golang"]
date: "2019-12-13T21:00:00+08:00"
toc: true
---

文章归档

天真的我一直以为文档归档就是文章分类的意思，折腾了一趟后，才知道文章归档就是把文章按照时间线列出。

Hugo默认是不支持生成归档文件的，需要自己实现。该主题已经实现了文章归档，只需要在新建content/archives/index.md文件，文件内容为：

---
title: "归档"
description: 人工智障、炼丹工程师
type: archives
---

title和description都可以换成你自己的，但是type必须是archives。

content/archives/index.md表示在content/archives/目录下的index.md文件

站内搜索

站内搜索默认是关闭的，如果需要使用需要以下步骤开启。

  1. 在config.toml中找到[params]配置段落，增加localSearch = true开启站内搜索
  2. 在content目录下新建search目录,并且在search目录中新建index.md文件，内容如下
---
title: "搜索"
description: "搜索页面"
type: "search"
---

注意此处的站内搜索，搜索只支持的是搜索标题，并不支持搜索文章内的内容关键词，以后写文章起标题要好好起。。。
